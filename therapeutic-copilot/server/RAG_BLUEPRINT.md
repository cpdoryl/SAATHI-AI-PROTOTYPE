### Version: 1.0 | Date: 2026-03-08 | Status: In Progress

## PURPOSE
This blueprint defines the complete Retrieval-Augmented Generation (RAG) system
for SAATHI AI — from document ingestion to context-enriched AI responses.
Each clinic (tenant) gets its own isolated knowledge base.

---

## 1. RAG ARCHITECTURE

```
Clinic Admin uploads documents (CBT guides, FAQs, protocols)
        │
        ▼
[INGESTION PIPELINE]
  1. Document text extracted
  2. Chunked into 512-token segments with 50-token overlap
  3. Each chunk embedded → 384-dim vector (all-MiniLM-L6-v2)
  4. Upserted to Pinecone with metadata {tenant_id, source, chunk_index}
        │
        ▼
[QUERY PIPELINE — runs on every patient message]
  1. Patient message embedded → 384-dim vector
  2. Pinecone similarity search in tenant namespace (top-k=3)
  3. Top chunks returned as context strings
  4. Injected into LLM system prompt before inference
        │
        ▼
[LLM RESPONSE]
  Context-aware therapeutic response generated by Qwen 2.5-7B
```

## 4. DOCUMENT CHUNKING IMPLEMENTATION

**Current problem**: `ingest()` takes `content[:1000]` — no real chunking.

**Required implementation** (`services/rag_service.py`):
```python
def _chunk_text(self, text: str, chunk_size: int = 512, overlap: int = 50) -> list[str]:
    """
    Split text into overlapping chunks of ~chunk_size tokens.
    Uses character-based approximation (1 token ≈ 4 chars).
    """
    char_size = chunk_size * 4
    char_overlap = overlap * 4
    chunks = []
    start = 0
    while start < len(text):
        end = start + char_size
        chunks.append(text[start:end])
        start += char_size - char_overlap
    return [c for c in chunks if len(c.strip()) > 50]  # filter tiny chunks
```

**Updated `ingest()` to use chunking**:
```python
async def ingest(self, content: str, tenant_id: str, metadata: dict) -> dict:
    index = self._get_client()
    chunks = self._chunk_text(content)
    vectors = []
    for i, chunk in enumerate(chunks):
        embedding = self._embed(chunk)
        chunk_id = f"{metadata.get('source', 'doc')}_{tenant_id}_{i}_{uuid.uuid4().hex[:8]}"
        vectors.append({
            "id": chunk_id,
            "values": embedding,
            "metadata": {
                "text": chunk,
                "tenant_id": tenant_id,
                "source": metadata.get("source", "unknown"),
                "chunk_index": i,
                "total_chunks": len(chunks),
                **metadata,
            }
        })
    # Batch upsert (max 100 per call)
    for batch in [vectors[i:i+100] for i in range(0, len(vectors), 100)]:
        index.upsert(vectors=batch, namespace=tenant_id)
    return {"chunks_ingested": len(chunks), "tenant_id": tenant_id}
```

## 7. QUERY PIPELINE TUNING

### Current query parameters (in `rag_service.py`):
```python
results = index.query(
    vector=embedding,
    top_k=3,
    namespace=tenant_id,
    include_metadata=True,
)
```

### Tuning targets:
- `top_k=3` — retrieves 3 chunks. Increase to 5 for longer context models.
- Similarity threshold: filter results with `score < 0.75` (low relevance)
- Fallback: if no results for tenant namespace → query `"default"` namespace

### Improved query with threshold + fallback:
```python
async def query(self, query: str, tenant_id: str, top_k: int = 3) -> list[str]:
    embedding = self._embed(query)
    # Try tenant namespace first
    results = index.query(vector=embedding, top_k=top_k,
                          namespace=tenant_id, include_metadata=True)
    contexts = [m.metadata["text"] for m in results.matches if m.score >= 0.75]
    # Fallback to default if tenant has nothing relevant
    if not contexts:
        results = index.query(vector=embedding, top_k=top_k,
                              namespace="default", include_metadata=True)
        contexts = [m.metadata["text"] for m in results.matches if m.score >= 0.70]
    return contexts
```
