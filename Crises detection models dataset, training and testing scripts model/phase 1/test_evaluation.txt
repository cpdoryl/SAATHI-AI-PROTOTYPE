Interpretation:
---------------
The model is intentionally biased toward high-risk prediction.
This is due to:
• Aggressive class weighting (8x boost for high-risk)
• Safety threshold-based prediction (probability > 0.20 → high-risk)
• Recall-optimized early stopping during training

As a result:
• NO high-risk cases are missed (zero false negatives)
• Accuracy and precision are low due to acceptable false positives

This is EXPECTED behavior for safety-critical systems where:
• Missing a suicidal case is unacceptable
• False alarms are tolerated

Model Classification:
---------------------
This model functions as a "Safety Gate" or "High-Recall Filter".

Common use cases:
• Suicide prevention systems
• Medical triage
• Fraud detection
• Safety monitoring pipelines

Deployment Note:
----------------
This model is NOT intended to be user-facing as a final decision system.
Recommended production setup:

1. Stage 1 – Safety Gate (current model)
   • Ensures near-zero false negatives

2. Stage 2 – Risk Refinement Model

   • Improves precision and user experience
